mtk.ga.population module¶

Defines the Population class.

class mtk.ga.population.Population(*args)[source]¶

Bases: object

A container for Molecule objects.

Population instances can be nested.

In addtion to holding Molecule objects, the Population class can be used to create large numbers of these instances through the init_() class methods.

Molecule instances held by a Population can have their structures optimized in parallel through the optimize_population() method.

The EA is invoked by calling a number of methods of this class, such as gen_offspring(), gen_mutants() and select(). However, this class only implements container related functionality. It delegates EA operations to the Crossover, Mutation and Selection classes.

These classes are organised as follows. Each Population instance has a ga_tools attribute. This holds a GATools instance. The GATools instance is just a container. It holds a Crossover, Mutation and Selection instance. These instances deal with the Population instance they are held by and perform the various crossover, mutation and selection operations on it. Any functionality related to the EA should be delegated to these instances. The gen_offspring() and gen_mutants() methods can serve as a guide to how this should be done.

The only operations directly implemented by this class are those relevant to its role as a container. It supports all expected and necessary container operations such as iteration, indexing, membership checks (via the is in operator) as would be expected. Details of the various implementations and a full list of supported operations can be found by examining the included methods.

populations¶

list of Population instances – A list of other instances of the Population class. This allows the implementation of subpopulations or evolutionary islands. This attribute is also used for grouping molecules within a given population for organizational purposes.

members¶

list of Molecule instances – Held here are the members of the population which are not held within any subpopulations. This means that not all members of a population are stored in this attribute. To access all members of a population the generator all_members() should be used.

ga_tools¶

GATools – An instance of the GATools class. It stores instances of classes such as Selection, Mutation and Crossover, which carry out GA operations on the population.

__getitem__(key)[source]¶

Allows indexing to select members of the population.

Molecules held by the Population instance can be accessed by their index. Slices are also supported. These return a new Population instance holding the members with the requested indices. Using slices will return a flat Population instance, meaing no nesting is preserved.

The index corresponds to the order of members in all_members().

Parameters:key (int, slice) – An int or slice can be used depending on if a single members needs to be returned or a cllection of them.
Returns:
  • Molecule – If the supplied key is an int. Returns the Molecule instance with the corresponding index from all_members().
  • Population – If the supplied key is a slice. The returned Population instance holds members at the given indices in all_members().
Raises:TypeError – If the supplied key is not an int or slice.

__sub__(other)[source]¶

Removes members of other from the population.

Subtracting one population from another,

pop3 = pop1 - pop2

returns a new population, pop3. The returned population contains all molecules in pop1 excep those also found in pop2. This refers to all molcules, including those held within any subpopulations. The returned population is flat. This means any nesting in pop1 is not preserved.

The resulting population, pop3, will hold the same GATools instance as pop1.

Parameters:other (Population) – A collection of Molecule instances to be removed from the population, if held by it.
Returns:A flat population of Molecule instances which are in the population but not in other.
Return type:Population

calculate_member_fitness()[source]¶

Applies the fitness function to all members.

The fitness function is defined in the attribute GATools.fitness of the GATools instance held in the ga_tools attribute of the population.

The calculation will be performed serially or in parallel depending on the flag GATools.parallel. The serial version may be faster in cases where all molecules have already had their fitness values calcluated. This is because all calculations will be skipped. In this case creating a parallel process pool creates unncessary overhead.

Notes

This method will result in the modification of the MacroMolecule.unscaled_fitness attribute of molecules held by the population.

Returns:None

classmethod init_all(databases, bb_classes, topologies, macromol_class, num_cores=None, ga_tools=<mtk.ga.ga_tools.GATools object>, duplicates=False)[source]¶

Creates all possible molecules from a given set of databases.

All possible combinations of building blocks from databases and topologies from topologies are built. The initialization of a macromolecule in the population has the form

mol = macromol_class([bb1, bb2, ...], topology)

where bb1 is initialized from a molecule in databases[0], bb2 from a molecule in databases[1] and so on.

Parameters:
  • databases (list of str) – List of paths to directories, which hold molecular structure files of the building blocks.
  • bb_classes (list of type) –

    This list must be equal in length to databases. For each database provided in databases, a class used to initialize building blocks from that database is provided here. For example, if

    databases = ['/path/to/amines2f',
             '/path/to/aldehydes3f']

    then a valid bb_classes list would be

    bb_classes = [StructUnit2, StructUnit3]

    This means all molecules in the amines2f database are initialized as StructUnit2 objects, while all molecules in aldehydes3f are initialized as StructUnit3 objects.

  • topologies (list of Topology) – The topologies of macromolecules being made.
  • macromol_class (type) – The class of the MacroMolecule objects being built.
  • num_cores (int, optional) – The number of cores to use when building the molecules in parallel.
  • ga_tools (GATools, optional) – Stores the selection, mutation and crossover functions to be used on the population.
  • duplicates (bool, optional) – If True, duplicate structures are not removed from the population.
Returns:

A population holding all possible macromolecules from assembled from databases.
Return type:

Population

Examples

If the name of the functional group needs to be provided to the building blocks, a lambda function can be used.

dbs = ['/path/to/db1', 'path/to/db2']
bb_classes = [lambda x: StructUnit2(x, 'aldehyde'),
              lambda x: StructUnit3(x, 'amine')]
tops = [Linear("AB", [0, 0], 6)]
pop = Population.init_all(dbs, bb_classes, tops, Polymer)
classmethod init_diverse(macromol_class, building_blocks, topologies, size, ga_tools)[source]¶

Assembles a population of MacroMolecule.

All molecules are held in the members.

From the supplied sublists of building blocks, a random molecule is selected to initialize a MacroMolecule per sublist. The next molecule selected from the same sublist is the one most with the most different Morgan fingerprint. The next molecule is picked at random again and so on. This is done until size MacroMolecule instances have been formed.

Parameters:
  • macromol_class (type) – The class of MacroMolecule to be assembled.
  • building_blocks (list) –

    A list of the form

    building_blocks = [[StructUnit2(), StructUnit2(), ...],
                   [StructUnit3(), StructUnit3(), ...],
                   [StructUnit2(), StructUnit2(), ...]]

    To assemble a new MacroMolecule, a StructUnit is picked from each of the sublists in building_blocks. The picked StructUnit instances are then supplied to the macromolecule:

    macro_mol = MacroMolecule([pick1, pick2, pick3],
                          Topology())

    The order of picked StructUnit instances corresponds to the order of the sublists.

  • topolgies (iterable of Topology) – An iterable holding topologies which should be randomly selected during initialization of MacroMolecule.
  • size (int) – The size of the population to be initialized.
  • ga_tools (GATools) – The GATools instance to be used by created population.
Returns:

A population filled with generated molecules.
Return type:

Population

remove_duplicates(between_subpops=True, key=<built-in function id>, top_seen=None)[source]¶

Removes duplicates from the population and preserves structure.

The question of which molcule is preserved when duplicates are removed is difficult to answer. The iteration through a population is depth-first, so a rule such as “the molecule in the topmost population is preserved” is not the case here. Rather, the first molecule found is preserved.

However, this question is only relevant if duplicates in different subpopulations are being removed. In this case it is assumed that it is more important to have a single instance than to worry about which subpopulation it is in.

If the duplicates are being removed from within subpopulations, each subpopulation will end up with a single instance of all molecules held before. There is no “choice”.

Parameters:
  • between_subpops (bool, optional) – When False duplicates are only removed from within a given subpopulation. If True, all duplicates are removed, regardless of which subpopulation they are in.
  • key (callable, optional) – Two molecules are considered the same if the values returned by key(molecule) are the same.
Returns:

None
Return type:

NoneType
